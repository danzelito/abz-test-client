Hello. Welcome to my abz-test-task application. This application is only for demonstration purposes and should not in any way to be used in production

This is a very simple client that allows you to connect to REST API Server. For a connection to be successfull you need to follow next steps:

  1. Get Token. The link is in Navigation Bar. It is valid for 40 minutes. After Token expiration you will be prompted to Get Token again to Continue on these Pages: Regitration, All Users and One User. Links are in the Navigation Bar
  2. Route protection is not implemented. So despite Registration, All Users and One User links are disabled by default, you can go to them inserting all-users.html, register.html and one-user.html respectively
  3. After getting a Token link for registration will become available. Fill out form and remember All fields are required
  4. Next You can get list of users directly from All Users link in Navigation Bar. You will get the List of 45 FAKE Users by default + users You created in the registration page. They are divided by 6 users on the page according to test task. Pagination seems to work. I tried to adapt it to different numbers of users on the client.
  5. data-generator.js is a data seeder and generator for fake users. It is in abz_test_task package Keys are:
    --deleteUserData => Removes all users from database
    --importUserData => Creates fake users in database. 45 by default.
    --deletePositionData => Removes all positions from database
    --importPositionData => Creates positions in database. 4 by default.
    Example(From root directory of the project): node ./helpers/data-generator.js --deleteUserData
  6. Technologies used:
    REST API => Node.js/Express/MongoDB; Modules: jsonwebtoken, mongoose, multer, tinify, morgan, express-validator, dotenv
  7. Note that in the Browser itself you cannot interact with RSET API because token is missing. I included abz_test_task.postman.json file in the helpers directory of the abz_test_task module with possible requests. Maybe it comes in handy.
